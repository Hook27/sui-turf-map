Hoe je via Claude wijzigingen doorvoert — na de module-splitsing · v3.43+

De situatie na splitsing

Na de splitsing bestaat de tool uit meerdere bestanden in de repo:

sui-turf-map/ ├── index.html ← alleen HTML-structuur + <script src>-verwijzingen ├── globals-stub.js ← jouw spiekbriefje voor context (niet live) └── modules/ ├── core.js ← data laden, globals ├── canvas.js ← kaart tekenen ├── garrison.js ← garrison modal ├── analyzer.js ← battle analyzer └── … ← alle andere modules
Voordeel: je stuurt Claude nu alleen het kleine relevante bestand mee, in plaats van de hele index.html van 4400 regels.

Stap-voor-stap werkwijze

1
Zoek op: welke module heeft dit?
Gebruik de Module-tabel hieronder. Zoek op de functienaam, het onderdeel van de UI, of de feature die je wilt wijzigen. De tabel geeft je de bestandsnaam.
2
Open die module in GitHub Desktop / teksteditor
Ga naar modules/<naam>.js in de repo-map. Kopieer de volledige inhoud van dat bestand.
3
Stuur Claude: globals-stub + de module + je verzoek
Zie het promptsjabloon hieronder. Upload globals-stub.js en de module als bijlage, of plak de inhoud.
Nooit de hele index.html meesturen — dat kost 30K tokens. De stub + één module = 2–5K tokens.
4
Ontvang het gewijzigde modulebestand
Claude geeft alleen het gewijzigde .js-bestand terug. Download het en vervang het bestand in de repo-map.
5
Commit & push via GitHub Desktop
Commit-bericht in het Engels. Test lokaal (Python HTTP-server) voor je pusht.
Vergeet niet: versienummer ophogen in index.html bij elke release.

Kopieer dit en vul de gele stukken in:

Snelzoekgids — ik wil…

Uiterlijk / layout wijzigen

  • Kleur knop/tekst→ CSS in index.html
  • Knop toevoegen toolbar→ HTML in index.html
  • Nieuwe modal→ HTML + CSS in index.html
  • Spelersrij anders→ sidebar.js
  • Tile kleur op kaart→ canvas.js
  • Mobiele weergave→ CSS in index.html

Functionaliteit wijzigen

  • Garrison tabs→ garrison.js
  • Aanvalsadvies→ sim.js
  • Battle Analyzer→ analyzer.js
  • Soft Targets logica→ intel.js
  • Leaderboard→ leaderboard.js
  • Routeberekening→ route.js

Data / blockchain

  • Data laden/verwerken→ core.js
  • Recall uitvoeren→ wallet.js
  • 24h overlay→ battle-hist.js
  • Spelershistorie→ leaderboard.js
  • Raids-data→ gar-history.js
  • Exporteren→ export.js

Overlays op de kaart

  • Ghost mode→ ghost.js
  • 24h veranderingen→ battle-hist.js
  • Route overlay→ route.js + canvas.js
  • Tooltip bij hover→ canvas.js (muis) / input.js (touch)
  • Buurpopup→ canvas.js
  • Minimap→ canvas.js

Valkuilen

CSS of HTML wijzigen? Die zitten nog in index.html zelf (niet in een aparte module). Stuur dan: globals-stub.js + alleen de CSS-sectie (r.12–327) of HTML-sectie (r.328–856) — niet de hele file.
Wijziging raakt meerdere modules? Stuur alle betrokken modules mee én vraag Claude om elke module apart als download te leveren.
Twijfel je? Zoek de functienaam op in de module-tabel, of vraag Claude: "In welke module zit [functie/feature]?" met alleen globals-stub.js als bijlage. Dat kost bijna geen tokens.
globals-stub.js bijwerken? Doe dit alleen als er nieuwe globale variabelen of functies bijkomen (na een grotere release). Vraag Claude dan om de stub bij te werken na de andere wijzigingen.
